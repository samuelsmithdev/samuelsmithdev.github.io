Project Idea

Overview

The project idea is a mobile App called “Eating Green” that finds the nearest vegan foods to the user's location. It also contains a function to take an image of a food and respond if whether the item is vegan or not so that users can make the correct decisions while doing grocery shopping without having to look through the ingredients list on an item.

Motivation

According to the national nutrition survey VeganAustralia.org.au estimates that 2% of Australia's population is vegan (Vegan FAQ, 2021). Which comes to approximately 500,000 vegans having to struggle through shopping centers and take away apps to find out what foods they are able to eat that will keep them on track with their constricted diet. Using the power of the modern-day mobile phone, googles Cloud Vision API and camera object detection we can make the lives of vegan Australians much more efficient.

Description

Android and iOS mobile device store users can download the Eating Green app from the app store. From there when they open the app, they are greeted by a welcome screen along with a google services login screen (google Identity API). The app then asks for your device's location and camera permissions. From here you have a screen with two buttons and a small settings cog icon at the top of the screen with the option to sign out of google services and contact information to the developer so that users can have direct contact to support if anything goes wrong with the app. One of the main buttons takes you to a maps screen which highlights all the nearby vegan eating locations within a radius of the user's selection. This is viewable in list format or maps format. Users can select a location and either upvote or downvote it and leave reviews. This gives other local vegans an indication on whether a restaurant is worth checking out. If the location isn't vegan but has vegan options the app will indicate so. Users also have the ability to recommend certain items and even recommend certain combination of an item (e.g., “I recommend ordering this burger with vegan mayonnaise.”). The other main button is an item scanning function. The user places their phones camera over an object. The object is then scanned and it responds with a green tick over the item if its vegan or a red cross if it's not. If the item has trouble being scanned into the API it comes back with an “Error. Please try again or contact support.". If an item comes back as being vegan users, then get a selection of recommendations under “other shoppers also loved these”. The recommendation list consists of items that other users had scanned in and upvoted.

Tools and Technologies

Using Visual studio, we can use Microsoft's Xamarin CLI we can build the apps front end. From here we need something to run our map's function, a user database, user recommendations and the camera item recognition. Google has many handy APIs for app building. We can use Google Identity API for logins, Google Maps Platform API for our maps function and Google Cloud Vision API for the item scanning functions. Using WAMP server we can store, send and receive data for our app functions over the web. 

Skills Required

We will need Visual Studio with Xamarin installed and a WAMP server installed.The previously mentioned technologies require programming skills and knowledge in languages such as MySQL, PHP, C# and XAML. With these 4 languages we can control the WAMP server and Xamarin development along with the use of the Google APIs. These skills can be easily learnt with online courses.

Outcome

If this project were to be successful the vegan population of Australia will have their diet and lifestyle simplified giving them time to focus on other things rather than spending hours researching how to live their restrictive lifestyles.
